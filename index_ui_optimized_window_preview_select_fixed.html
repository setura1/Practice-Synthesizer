軽量MIDI学習シンセ v2 先にレンダリング → あとで再生

重いリアルタイム合成を避け、MIDIを先にPCMへレンダリングしてから再生します。再生中はリサジュー表示、ボード線図風の片対数FFT、短時間FFTを表示します。擬似リバーブ/ディレイによる空間系、Karplus-Strong による物理モデリングも加え、シンセサイザーとDTMの学習に使えるようにしています。

この版の考え方

機能は削らずに残し、普段よく触る項目だけを前面に出しました。上級DSPや理論表示は折りたたみの中にまとめ、必要なときだけ開ける構成です。

  • ADSR、cutoff、delay、feedback、density
  • レンダリング、再生、WAVダウンロード

上級音作りとDSP設定を表示

ここには、LFO波形、Velocity to Filter、Drive、Chorus、FM、ノイズ、ウェーブテーブル、コンプレッサー、Parametric EQ、オーバーサンプリング、Convolution Reverb など、基本操作より一段上の項目をまとめています。初回は閉じたままでも再生できます。

基本の学習では、まず ADSR・cutoff・delay 系だけで十分です。上級音作りを開くと、LFO、FM、ウェーブテーブル、コンプレッサー、EQ、Convolution Reverb まで同じレンダリング系の中で比較できます。Karplus-Strong String は短い雑音を遅延ループで循環させ、弦をはじく近似モデルとして学べます。

チャンネル別音色を表示 上級

1つのMIDIにメロディ、伴奏、ベース、ドラムが混ざっているときに、チャンネルごとに音色を分けて再現度を上げるための設定です。単一音色で構造を見たいときは閉じたままで構いません。

MIDI読込後に各チャンネルの音色を切り替えられます。標準MIDIの Program Change を簡易対応し、Auto を選ぶと GM 的な近似音色へ割り当てます。ドラムチャンネルは簡易ドラム合成を使います。

使い方と表示の読み方

基本の流れ

  1. MIDIを選ぶ
    .mid / .midi を読み込みます。読込後はノート列がピアノロールに展開され、必要ならチャンネル別音色も確認できます。
  2. 音色・サンプルレート・polyphony を決める
    まずは軽い設定で音の全体像をつかみ、必要なら後から高品位側へ上げるのが扱いやすいです。
  3. ADSR と filter を軽く調整する
    Attack / Decay / Sustain / Release と cutoff / Q を触ると、立ち上がり・余韻・明るさがすぐ変わります。
  4. レンダリングして再生する
    このページは先にPCMを計算してから再生するので、再生中は比較的軽く、耳と表示を対応づけやすい構成です。
  5. 表示を順番に読む
    全体波形 → 再生位置周辺の波形 → FFT / STFT → ピアノロール → ADSR / フィルタ包絡の順で見ると理解しやすいです。
  6. 必要なら WAV 保存や上級DSPを見る
    WAV 保存はレンダ済みPCMをそのまま保存します。上級DSPは普段は閉じたままで問題ありません。

各表示の意味

  • 全体波形 : 曲全体のエネルギーの流れ。サビや余韻の密度が分かります。
  • 再生位置周辺の波形 : いま鳴っている局所波形。drive や倍音の増え方を見やすい表示です。
  • リサジュー表示 : 左右チャンネルまたは dry / wet の関係。位相差やステレオの広がりを読みます。
  • 線形FFT : 基音と倍音の位置関係をそのまま読む表示です。
  • Bode風FFT : 低域から高域までの帯域バランスを見やすくした表示です。
  • 短時間FFT : 時間変化を追う表示。アタックの鋭さやドラムの瞬間成分が分かります。
  • ピアノロール : 横が時間、縦が音高。メロディ、伴奏、ベースの役割分担を確認できます。
  • 鍵盤 : 単音試聴。FFT と合わせると、1音ごとの倍音構造を確かめやすいです。
  • ADSR / フィルタ包絡 : 音量と cutoff が時間でどう変わるかを読みます。

最初に触ると効果が分かりやすい項目

  • 音色を切り替える
  • Attack と Release を変える
  • cutoff と Q を変える
  • LFO と Velocity to Filter を少しだけ上げる
  • 最後に Chorus / EQ / Compressor などを足す
学習メモ / 基本から上級へ
  • 周波数 f = 440 × 2^((n-69)/12) でMIDI番号と音高がつながります。
  • 矩形波やパルス波は倍音が多く、ファミコン風の輪郭を作りやすいです。
  • FFT は周波数成分、STFT はその時間変化を見るための道具です。
  • ADSR は音量の時間変化、フィルタ包絡は音色の時間変化です。
  • Velocity to Filter を上げると、強打時に高域が増えて音が明るくなります。
  • PolyBLEP は square / pulse の不連続点をなめらかにし、折り返し雑音を抑えます。
  • チャンネル別音色を使うと、メロディ・伴奏・ベースを別の役割で聴き分けやすくなります。
実装メモ / なぜこの構成か
  • 外部ライブラリなしで、GitHub Pages 上でも動かしやすい構成です。
  • レンダリング後に再生するため、再生中の負荷を低減できます。
  • 再生はステレオ化し、空間系とリサジュー表示に利用しています。
  • 最大同時発音数で負荷を抑え、重いMIDIでも扱いやすくしています。
  • ローパスや一部のエフェクトはレンダ後適用を使い、動作の安定を優先しています。
  • WAV 保存は既に計算した PCM を Blob 化して保存するため、自然な拡張です。
数式と理論解説を開く

この欄はレンダリング後に、現在の音色・フィルタ・解析設定に合わせて更新されます。普段は閉じたままでよく、式を追いたいときだけ開けば十分です。

MIDIを読み込んでレンダリングすると、ここに現在の設定に対応した数式解説が表示されます。発振器、ADSR、フィルタ、LFO、FM、EQ、Compressor、FFT / STFT、z平面、Bode Plot などをまとめて確認できます。
